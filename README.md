# 🎬 Movie Archive

آرشیو فیلم شخصی، آفلاین و محلی برای ویندوز — پوشه‌ی فیلم‌هاتون رو اسکن می‌کنه، خودکار با [TMDB](https://www.themoviedb.org) تطبیق می‌ده، پوستر/بازیگر/امتیاز/خلاصه‌داستان رو یک‌بار برای همیشه توی یک دیتابیس SQLite محلی کش می‌کنه، و رابط کاربری فارسی/انگلیسی با تعویض خودکار جهت راست‌به‌چپ/چپ‌به‌راست داره.

ساخته‌شده با JavaFX (جاوا ۲۱) — بدون سرور، بدون کلود، همه‌چیز روی سیستم خودتون.

---

## ✨ ویژگی‌ها

### تشخیص و تطبیق فیلم
- تشخیص هوشمند نام فیلم و سال از روی نام پوشه/فایل، حتی با نام‌های شلوغ و پر از برچسب‌های کیفیت/گروه ریلیز (`The.Matrix.1999.1080p.BluRay.x264-YIFY` → `The Matrix (1999)`)
- تطبیق خودکار با TMDB، با تشخیص هوشمند «نتیجه مطمئن» در برابر «نیاز به بررسی دستی» (چند نتیجه‌ی نزدیک)
- **صف بازبینی**: لیست متمرکز همه‌ی فیلم‌های تطبیق‌نیافته، با اصلاح تک‌تک یا تطبیق دسته‌جمعی
- تطبیق‌های موازی با Virtual Threads (جاوا ۲۱) برای سرعت بالا حتی با چند هزار فیلم
- چندپوشه‌ای: چند پوشه‌ی آرشیو مختلف رو ثبت و با یک کلیک همه رو اسکن کنید

### داده و کش محلی
- پوستر، خلاصه‌داستان، بازیگران، کارگردان، ژانر/کلیدواژه، امتیاز، کشور سازنده، رده‌بندی سنی (MPAA)، و لینک IMDb — همه یک‌بار دریافت و برای همیشه در SQLite ذخیره می‌شن
- **بکاپ کامل**: دیتابیس + همه‌ی پوسترها در یک فایل zip، بدون نیاز به دریافت مجدد از TMDB بعد از بازگردانی
- Export/Import به JSON برای بکاپ‌های سبک و قابل‌دیف
- بروزرسانی متادیتای فیلم‌های قبلاً تطبیق‌یافته (تکی یا دسته‌جمعی) بدون دانلود دوباره‌ی پوستر

### رابط کاربری
- عملکرد روان حتی با هزاران فیلم، به لطف گرید و لیست مجازی‌سازی‌شده (فقط ردیف‌های قابل‌مشاهده رندر می‌شن)
- سه نمای نمایش: فشرده، پوستر بزرگ، لیست با جزئیات
- سایدبار فیلترهای یکپارچه: ژانر و تگ (چندانتخابی، منطق OR - یعنی فیلم‌هایی که حداقل یکی از موارد انتخاب‌شده رو داشته باشن نشون داده می‌شن)، علاقه‌مندی/دیده‌شده، و فیلتر پیشرفته (بازه‌ی سال/امتیاز/مدت‌زمان، کشور، رده‌بندی سنی) — همه زنده و بدون نیاز به دکمه‌ی «اعمال»
- نمایش تعداد فیلم‌های در حال نمایش بالای لیست (و در صورت فعال‌بودن فیلتر، تعداد فیلترشده از کل)
- جستجوی سراسری در نام، کارگردان، بازیگر، و خلاصه‌داستان
- کلیک روی نام بازیگر/کارگردان یا هر تگ در پنل جزئیات، بلافاصله جستجو/فیلتر رو به همون سمت می‌بره
- فیلدهای شخصی (دیده‌ام / علاقه‌مندی) مستقل از داده‌ی TMDB
- نوع نمایش، مرتب‌سازی، و اندازه/حالت پنجره بین اجراها حفظ می‌شن

### دوزبانگی (فارسی / انگلیسی)
- کل رابط کاربری، خلاصه‌داستان، کشور، و ژانرها بین فارسی و انگلیسی سوییچ می‌کنن
- جهت راست‌به‌چپ/چپ‌به‌راست به‌صورت خودکار با زبان هماهنگ می‌شه
- خلاصه‌داستان و نام کشور، هر دو نسخه (فارسی و انگلیسی) به‌صورت جداگانه ذخیره می‌شن تا سوییچ زبان نیاز به دریافت مجدد از TMDB نداشته باشه

### اتصال و دسترسی
- پشتیبانی از پروکسی HTTP برای شرایطی که TMDB مستقیماً در دسترس نیست

## 🚀 شروع سریع

### پیش‌نیازها
- JDK 21+
- Maven
- یک [کلید API رایگان TMDB](https://www.themoviedb.org/settings/api) (نوع «API Read Access Token»)

### ساخت و اجرا

```bash
git clone https://github.com/mehranirandegani/local-movies-archive-manager.git
cd local-movies-archive-manager/parser
mvn clean javafx:run
```

اولین بار که برنامه بالا میاد:
1. از دکمه‌ی **تنظیمات ← عمومی**، زبان دلخواه رو انتخاب کنید.
2. از تب **TMDB و پروکسی**، کلید API خودتون رو وارد کنید.
3. از دکمه‌ی **«افزودن و اسکن پوشه آرشیو...»**، پوشه‌ی فیلم‌هاتون رو انتخاب کنید.

برای بسته‌بندی به یک jar مستقل:

```bash
mvn clean package
```

### ساخت نصب‌کننده‌ی مستقل ویندوزی (بدون نیاز به نصب جاوا)

اگه می‌خواید برنامه رو به یه فایل `.exe` قابل‌نصب تبدیل کنید که خودش یه JRE مخصوص به خودش داره (کاربر نهایی اصلاً نیازی به نصب جاوا نداره)، از `jpackage` (که از JDK 14 به بعد داخل خود جاوا هست) استفاده کنید:

**پیش‌نیاز (فقط روی سیستمی که نصب‌کننده رو می‌سازه، نه سیستم کاربر نهایی):**
- JDK 17+ و Maven (که برای توسعه هم لازم داشتید)
- [WiX Toolset v3](https://wixtoolset.org/) — `jpackage` برای ساخت `.exe`/`.msi` روی ویندوز بهش نیاز داره

**اجرا:**

```bash
build-installer.bat
```

این اسکریپت اول با Maven یه jar کامل (شامل تمام وابستگی‌ها) می‌سازه، بعد با `jpackage` یه نصب‌کننده‌ی ویندوزی توی پوشه‌ی `dist` تولید می‌کنه. اگه WiX نصب نکردید، توی `build-installer.bat` مقدار `--type exe` رو به `--type app-image` تغییر بدید تا به‌جای نصب‌کننده، یه پوشه‌ی قابل‌اجرا (که می‌تونید zip و به اشتراک بذارید) بسازه.

> نکته: چون این اسکریپت برای پلتفرم ویندوز تنظیم شده (وابستگی‌های JavaFX با classifier `win` در `pom.xml` قفل شدن)، برای ساخت روی مک/لینوکس باید اون classifier رو به `mac`/`linux` تغییر بدید.

## ⚙️ پیکربندی

### پروکسی (در صورت فیلترینگ/تحریم)

اگه TMDB مستقیماً در دسترس نیست، از **تنظیمات ← TMDB و پروکسی**، پروکسی HTTP رو فعال و آدرس/پورت وارد کنید.

> ⚠️ باید پورت **HTTP** ابزار VPN/پروکسی‌تون باشه، نه پورت SOCKS5 — کتابخانه‌ی شبکه‌ی جاوا (`java.net.http.HttpClient`) از SOCKS پشتیبانی نمی‌کنه. اکثر ابزارهای رایج (V2rayN، Clash، و مشابه) یه پورت HTTP جدا از SOCKS دارن.

### محل ذخیره‌سازی

تنظیمات، دیتابیس، و کش پوسترها در `%APPDATA%\MovieArchive` (ویندوز) ذخیره می‌شن. کلید API TMDB عمداً در فایل تنظیمات جدا از دیتابیس نگه‌داری می‌شه تا هنگام اشتراک‌گذاری/بکاپ آرشیو، لو نره.

```

  pom.xml
  src/main/java/com/moviearchive/
    parser/       تشخیص نام فیلم و سال از روی نام فایل/پوشه
    scan/         پیمایش پوشه‌های آرشیو
    tmdb/         کلاینت TMDB، منطق تطبیق، پردازش موازی
    db/           دیتابیس SQLite، Export/Import، بکاپ کامل
    media/        دانلود و کش پوستر
    model/        مدل داده‌ی فیلم
    config/       تنظیمات برنامه (پروکسی، زبان، کلید API)
    ui/           رابط کاربری JavaFX (پنجره‌ی اصلی، دیالوگ‌ها، متن‌های دوزبانه)
  src/main/resources/
    schema.sql    اسکیمای دیتابیس + migration خودکار
    styles.css    تم رابط کاربری
test/
  TestParser.java تست پارسر نام فیلم روی نمونه‌های واقعی
```

## 🌐 نکته‌ی مهم قانونی (الزامی)

این پروژه از [TMDB API](https://www.themoviedb.org/documentation/api) برای دریافت اطلاعات فیلم استفاده می‌کنه. طبق شرایط استفاده‌ی TMDB، ذکر این جمله الزامیه:

> **This product uses the TMDB API but is not endorsed or certified by TMDB.**

این پروژه توسط TMDB تأیید یا پشتیبانی نمی‌شه. اگه قصد انتشار عمومی یا توزیع گسترده‌ی این اپ رو دارید، لطفاً [شرایط استفاده‌ی TMDB](https://www.themoviedb.org/api-terms-of-use) و [راهنمای برند/لوگو](https://www.themoviedb.org/about/logos-attribution) رو هم مطالعه کنید (استفاده از لوگوی TMDB در رابط کاربری طبق همون راهنما اختیاری ولی توصیه‌شده‌ست).

این پروژه هیچ داده‌ای از IMDb دریافت یا ذخیره نمی‌کنه؛ فقط یک لینک مستقیم به صفحه‌ی عمومی IMDb هر فیلم (بر اساس شناسه‌ی عمومی `imdb_id` که خود TMDB ارائه می‌ده) برای دسترسی راحت‌تر کاربر فراهم می‌کنه.

## ⚠️ محدودیت‌های شناخته‌شده

- **کلیدواژه‌ها همیشه انگلیسی هستن:** TMDB برای ژانرها (Action, Drama, ...) ترجمه‌ی فارسی داره، ولی برای کلیدواژه‌های آزاد (based on novel, time travel, ...) ترجمه‌ای ارائه نمی‌ده.
- **امتیاز/محبوبیت، مال TMDB هستن نه IMDb:** این دو عدد با امتیاز/محبوبیت واقعی IMDb یکی نیستن (هرچند معمولاً نزدیکن). برای عدد خالص IMDb باید OMDb API رو جداگانه اضافه کرد.
- **تعویض زبان، اپ رو ری‌استارت می‌کنه:** به‌جای بازسازی زنده‌ی رابط کاربری، تعویض زبان از تنظیمات باعث بسته‌شدن و باز شدن مجدد پنجره‌ی اصلی می‌شه (تنظیمات و داده‌ها حفظ می‌شن).
- **فقط فیلم، نه سریال:** این اپ از پایه برای آرشیو فیلم طراحی شده؛ سریال مدل داده و API متفاوتی نیاز داره.
- **فقط ویندوز تست شده:** به‌خاطر استفاده از `Desktop.open()` برای پخش فایل و مسیر `%APPDATA%`، احتمالاً نیاز به تنظیم جزئی برای macOS/Linux داره.
